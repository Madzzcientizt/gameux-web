        Overview

          Sekai Architecture:

        • Unity Mobile Client
        • Lane-Perspective Rhythm Field
        • Character + Live Service Layer

        Hatsune Miku: Colorful Stage! (Project Sekai, 2020) is a mobile rhythm game built around touch gestures and character-driven progression. Core songs are played in 2D rhythm charts while the broader product loop includes card collection, unit building, and event-based score goals.

        From an HCI perspective, Sekai is a strong case study in mobile ergonomics: it must preserve timing precision without the mechanical certainty of physical keys. The interface balances readability and spectacle through lane perspective, configurable note speed, lane offset, and background dimming options that reduce visual noise.

        • Cognitive Load
          Medium to High
          Players track lane depth, gesture type shifts, and combo-state feedback in real time.
        • Response Time
          Touch-latency constrained
          Performance depends on device touch sampling, frame pacing, and player calibration settings.
        • Primary Input Surface
          Capacitive glass
          No physical key travel, so visual/auditory judgement feedback becomes essential for confidence.

        UI Components

        Sekai's core UI follows a "dense but legible" philosophy: only information needed for timing, survival, and score optimization remains persistent during songs; secondary systems are deferred.

        [ ELEMENT_01 ] Dynamic Lane Scaling

        Project Sekai uses perspective-driven lane scaling and variable note widths that funnel into a straight judgement line, improving readability on small touch screens without changing core timing logic.

        [ ELEMENT_02 ] Directional Flick Arrows

        Specialized gestures (Flicks) require the user to drag fingers across the glass, adding kinesthetic energy to the otherwise flat interaction format. Directional cues reduce ambiguity and improve first-read comprehension in dense charts.

        [ ELEMENT_03 ] Variable Width Lane Bars

        Notes vary from 1-lane to 6-lane widths dynamically. This uses visual weighting to signify note importance and input leniency, especially during simultaneous lane pressure.

        Persistent In-Song HUD Channels

        1. Judgement & Combo

          Immediate timing labels and combo continuity expose precision and error cost instantly.

        2. Score & Skill State

          Score growth, multiplier behavior, and temporary skill activations communicate run quality.

        3. Life / Clear Safety

          The life gauge converts timing mistakes into visible clear-risk, shaping player pacing decisions.

        User Interaction

        The UX flow is built specifically for capacitive glass, mitigating the lack of tactile hardware feedback through kinesthetic empathy.

        The Capacitive Loop

        1. Read Depth

          Track incoming note groups along depth cues and approach timing.

        2. Tap/Flick

          Execute tap or directional flick according to note iconography.

        3. Trace

          Maintain hold/slide continuity while preparing for simultaneous side-lane inputs.

        4. Validate

          Use judgement + combo feedback to micro-correct tempo and hand placement.

        Multi-Modal Feedback Matrix

        • Synthetic Haptic Rhythms

          Optional vibration cues can reinforce contact timing on mobile devices, partially replacing physical key feedback.

        • Background Dimming

          Players can reduce MV/3D visual intensity to prioritize note readability during high-density passages.

        HCI Principles

        Finger Occlusion Correction

        A critical HCI failure in mobile apps is 'Anatomical Occlusion' where the finger hides the UI. Sekai mitigates this by rendering judgement feedback above the primary hit zone, ensuring feedback is visible.

        Glass Friction UX

        Tactile Benefit Interaction Friction
        Gestural mapping

        Tap, hold, flick, and slide gestures map directly to visual note semantics, lowering interpretation overhead after practice.
        Kinetic Drag

        Finger moisture, screen protectors, and unstable grip can reduce slide consistency, especially on long chained traces.

        Visual Hierarchy & Complexity Management

        Sekai employs strict visual prioritization to prevent cognitive overload: during active gameplay, only notes and the judgement line persist as primary focus. Secondary UI (score, life bar, skill icons) occupies the periphery with muted opacity. Song-specific MV (music video) background rendering can be disabled or dimmed to reduce visual competition with lane content, a critical feature for high-difficulty charts where 10+ simultaneous notes appear.

        Adaptive Feedback & Skill Scaffolding

        Feedback Type Adaptive Mechanism
        Judgement Granularity

        Tutorial and early difficulties display only four tiers (PERFECT / GOOD / OK / MISS) to avoid analysis paralysis. Higher difficulties reveal the full five-tier system (PERFECT / COOL / FINE / SAFE / MISS), allowing skilled players to micro-optimize timing.
        Chart Progression

        Each song offers multiple difficulty levels that scaffold from simple 2-lane patterns (difficulty 1–5) to dense, lane-spanning arrangements (difficulty 25+). Players self-select their entry point, reducing frustration while maintaining mastery pathways.
        Auditory Reinforcement

        Judgement sounds vary by accuracy: PERFECT triggers a clean "ding," while MISS produces a distinct "thud." This non-visual feedback channel allows peripheral awareness of timing precision without diverting visual focus.
        Skill-Based Settings

        A practice mode lets players loop sections, adjust note speed & offset per-song, and disable MV rendering entirely. These "crutches" are removed during ranked play, but their availability during learning reduces initial barrier-to-entry.

        Strengths and Weaknesses Analysis

        System Strengths Positive_Vectors System Weaknesses Negative_Vectors
        01_ERGONOMICS
        Biomechanical Synergy

        Lane perspective and note scaling improve reach planning for thumbs across wide phone displays.
        01_HARDWARE
        Polling Rate Friction

        Lower touch sampling rates and unstable frame pacing can produce perceived timing inconsistency in dense charts.
        02_FLEXIBILITY
        Variable Width Tuning

        Allowing a single note to visually span 5 lanes ensures casual players never miss it, while keeping strict timing windows for pros.
        02_OCCLUSION
        Anatomical Occlusion

        Multi-finger tablet play can still block approach lanes, forcing memory-based input during fast alternations.
        03_FEEDBACK
        Synthetic Haptics

        Audio + optional vibration + judgement text create a layered confirmation stack that stabilizes player confidence.
        03_FRICTION
        Glass Moisture Drag

        Long slides remain sensitive to friction variation, producing avoidable breaks during endurance sections.

        Project Sekai expertly mitigates the innate flaws of capacitive touch screens—lack of haptics, anatomical occlusion, and fat-finger errors—by utilizing dynamic width scaling and synthetic vibration feedback to craft a tactile rhythm experience on flat glass.

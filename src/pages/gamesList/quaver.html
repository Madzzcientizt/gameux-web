        Overview

          Quaver Architecture:

        • Open Source (GitHub)
        • C# / FNA Framework

        Quaver (2020) is a community-driven, open-source vertical scrolling rhythm game (VSRG) heavily focused on hyper-competitive play in 4-Key and 7-Key modes.

        From a UI/UX standpoint, Quaver is the ultimate distillation of brutalist data transmission. It actively strips away "fun" aesthetic distractions (MVs, dancing avatars) to provide raw, unadulterated feedback: millisecond hit errors, real-time AP percentages, and ultra-high polling rate diagnostics.

        • Cognitive Load
          Extreme
          High note density reading (chordjacking/streaming 30+ taps per second).
        • Response Time
          ± 16ms
          Engine tuned to bypass bloated OS input threading.
        • Visual Priority
          Absolute
          100% of the screen real-estate is given purely to the note grid.

        UI Components

        [ ELEMENT_01 ] Strict Border Outlines

        The playfield is rigidly segmented by literal pixel-perfect columns. This allows the player's brain to chunk the visual data into distinct left-hand and right-hand processing blocks instantly.

        [ ELEMENT_02 ] Modular Skinning Interface

        Quaver allows for extreme personalization. Users can modify note-to-background contrast ratios and judgment-text positions. This grants 'UI Agency' to the player, allowing them to solve their own 'informational occlusion' issues.

        [ ELEMENT_03 ] Advanced Diagnostic Graphs

        A post-hit line graph showing active BPM density vs accuracy performance. This allows for high-level neuro-motor recalibration, letting the user identify exactly at what note-density threshold their cognitive processing loop begins to fail.

        User Interaction

        Interaction is purely competitive—translating extreme visual density into neuro-motor twitch responses via a zero-latency data pipeline. Every keystroke is a data point. Every microsecond of delay is failure.

        The Data Translation Loop

        1. Read Stream

          Process dense vertical note chunks arriving at high scroll speeds (1.0x–2.5x rate). The brain must parse 4–7 simultaneous input lanes while maintaining temporal awareness of incoming chains.

        2. Chunk Chords

          Group adjacent columns into singular rhythmic actions (left-hand vs right-hand mechanical units). Professionals treat hand-pairs as monolithic input units rather than discrete key presses.

        3. Actuate

          Depress physical mechanical switches with millisecond temporal precision. The input latency budget is fixed: ±16ms window. Outside this, judgment degrades from "Perfect" → "Great" → "Good" → Miss.

        4. Calibrate

          Real-time offset correction via the hit error bar. If drifting early, conscious micro-adjustments delay the next keystroke; if late, accelerate the tempo slightly. This closed-loop correction requires active decision-making mid-performance.

        Latency Architecture

        Quaver's performance obsession stems from a single truth: in rhythm games, latency IS the core mechanic. The engine prioritizes input polling at 1000Hz (every 1ms), rendering bypass bloated OS input threadpools entirely. This creates a direct neural-to-hardware pipeline where timing compensation is purely biological—the player's brain, not the computer's.

        • Mechanical Switch Haptics

          The physical keystroke generates tactile feedback (~0-5ms latency). This proprioceptive signal validates input registration before visual confirmation, acting as a low-latency checkpoint for motor memory.

        • Hit Feedback & Accuracy Display

          Judgment text (Perfect/Great/Good/Miss) appears immediately after keystroke (~16ms render latency). Combined with the error bar, players receive dual-channel feedback: audio confirmation + visual judgment + temporal offset visualization.

        Game Modes & Input Schemes

        4-Key Mode

        Left-hand chord (D+F) and right-hand chord (J+K), mapped to a single spacebar. This creates a "toggle" interaction model where players rapidly alternate between left and right hand positions, creating a rhythm-based coordination challenge.

        7-Key Mode

        Full piano-like layout (D, F, G, Spacebar, H, J, K). Dramatically increases pattern complexity by introducing true poly-rhythmic sequences where independent fingers must execute different temporal offsets simultaneously.

        Offset Calibration

        Quaver's UI provides a dedicated offset wizard allowing players to dial in equipment latency to the millisecond. This eliminates hardware as a variable—if you can't hit the timing window, it's a biological/skill limitation, not a technical one.

        HCI Principles

        Temporal Precision in Synchronized Movement

        The human neuro-motor system operates within hard timing windows. At high note densities, every keystroke must land within ±16 milliseconds of the intended beat, or the rhythm breaks. This creates a cascading error state: miss one note, lose temporal synchronization, miss the next note. The cognitive load is not choosing what to do—it's executing with millimeter-perfect mechanical timing.

        Visual cognition cannot be bypassed. The performer must read AND execute in real-time, essentially running two parallel processes—visual parsing and motor execution—to maintain sync across 200+ simultaneous decisions per minute.

        Cognitive Load: The Vision-Motor Loop

        How does a player maintain perfect timing across 30+ note-densities per second while reading upcoming patterns and adjusting swing mid-stream without system failure?

        Visual Stream Processing

        The base load of parsing falling notes in real-time. At 30+ notes/sec, individual attention becomes biologically impossible; the brain chunks patterns into texture recognition.

        Motor Execution Load

        The physical keystroke timing precision required. Professionals decouple conscious attention from finger motion—turning macro-level awareness into micro-level muscle memory.

        Temporal Drift Correction

        Real-time adjustment of swing when the player's timing deviates. This requires active monitoring of the hit error bar while maintaining focus on incoming patterns—dual-task processing at maximum capacity.

        UI Information Density

        The cognitive overhead of parsing HUD metrics (AP%, hit error, accuracy feedback) while maintaining rhythm focus. Minimalist design reduces this load by eliminating non-critical visual noise.

        Success in Quaver is not about 'musicality' or 'feel'—it is the successful management of cognitive bandwidth under extreme temporal pressure. The display is the visual input stream, the keyboard is the motor output interface, and the player's auditory-motor cortex is the processor. When these align perfectly, the human becomes a biological metronome.
